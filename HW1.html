Vocab Definitions/My Website Directory

  • HTTP
    • Definition: The way computers talk back and forth to each other on the internet
    • Official Documentation
  • HTML
    • Definition: A standardized system for tagging text files to achieve font, color, graphic, and hyperlink effects on WWW pages
    • Official Documentation
  • Element
    • Definition: Part of webpage that designates what the item will look like to the user.
    • Official Documentation
  • Tag
    • Definition: A tag is used for creating an element. The nameof the tag is what is included within the brackets.
    • Official Documentation
  • Block (CSS)
    • Definition: An HTML Element that appears on a new line, underneath the preceding element.
    • Official Documentation
  • Doctype
    • Definition: In HTML te required preamble found at the top of all documents. Its purpose is to prevent a browser from switching into "quirks mode" when rendering a document.
    • Official Documentation
  • Head
    • Definition: The part of an HTML document that contains metadata about that doucment(including author, description, and links to CSS or Javascript files that should be applied to HTML).
    • Official Documentation
  • Attribute
    • Definition: Extends a tag, changing its behavior or providing metadata.
    • Official Documentation
  • Boolean
    • Definition: In computer science, a logical data type that can only have the values true or false.
    • Official Documentation
  • DOM(Document Object Model)
    • An API that represents and interacts with an HTML document. It is the organization system by which a computer reads your coding.
    • Official Documentation
  • CSS(Cascading Style Sheets)
    • Definition: Language that controls how webpages look in the browswer, all about presentation!
    • Official Documentation
  • Property(CSS)
    • A characteristic whose associated value defines one aspect of how the browser shoudl display the element.
    • Official Documentaiton
  • Selector(CSS)
    • Part of a CSS rule that describes what elements in a document rule will match.
    • Official Documentation

Reflection

Coding is a whole lot harder than I thought it was going to be! This is first technology class I have taken since 7th grade when we used Scratch to move a digital ball from one side of the screen to the other.

I have learned that it is super important to follow a few coding practices:

  1. Always write your closing tag when you write your closing tag. It will save so much time in the long run when you are searching for a missing closing tag.
  2. Check how your code is appearing every five lines. This way you will spot a mistake before you get too far down on your code and you are more likely to be able to find your mistale.
  3. Get familiar with your computer. You will have to be able to find the work that you saved in your documents file and know which commands to use in order to swiftly make changes to your doucment.

I do find coding to be rewarding as it is satisfying to see a finished html page that you created in the way that you wanted to, especially when appling CSS. Coding can be tedious at times(especially when creating a long nested list), but it can be fun to learn and use new elements of HTML and CSS. I have found the "labs" in class to be helpful and engaging exercises and I am easily able to apply the new skills at that time. However, when I go to apply the new information on my homework I struggle to apply and upload it correctly. Hopefully with some more practice coding will become more intuitive!
